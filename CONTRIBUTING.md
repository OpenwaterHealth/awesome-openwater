# Contributing to Awesome Openwater

Thank you for your interest in contributing to this awesome list!

## Guidelines

### Quality Standards

- Only add resources you've personally used or verified
- Ensure resource is relevant to Openwater development
- Keep descriptions concise (1-2 sentences)
- Check that all links work
- Follow alphabetical ordering within sections

### Format

**For list items:**
```markdown
- [Resource Name](link) - Brief description of what it is and why it's useful.
```

**For categories with subsections:**
```markdown
### Category Name

- [Item 1](link) - Description.
- [Item 2](link) - Description.
```

### How to Contribute

1. **Fork** the repository
2. **Create a branch**: `git checkout -b add-resource-name`
3. **Add your resource** to the appropriate section
4. **Commit**: `git commit -m "Add [resource name]"`
5. **Push**: `git push origin add-resource-name`
6. **Submit** a pull request

### What to Add

**Good additions:**
- Tutorials and guides you've created
- Research papers using Openwater platforms
- Tools and libraries you've developed
- Community projects you've built
- Conference talks or presentations
- Educational resources
- Development tools that work well with Openwater

**Please don't add:**
- Resources not directly related to Openwater
- Commercial products without clear educational/development value
- Broken or unverified links
- Duplicate resources
- Self-promotional content without real value

### Sections

Add your contribution to the most appropriate section:

- **Official Resources** - Official Openwater materials only
- **Platforms** - Core Openwater platforms (OpenLIFU, OpenMOTION)
- **Getting Started** - Beginner guides and tutorials
- **Tutorials** - Step-by-step walkthroughs
- **Research Papers** - Academic publications
- **Tools & Libraries** - Software tools
- **Community Projects** - Projects built by community
- **Conference Talks** - Presentations and videos
- **Hardware** - Hardware designs and resources
- **Clinical Applications** - Medical use cases
- **Development Tools** - Dev environment tools
- **Educational Resources** - Learning materials
- **Community** - Community resources and events

### Pull Request Process

1. Ensure your addition follows the format guidelines
2. Check that all links work
3. Verify alphabetical ordering
4. One resource per pull request (easier to review)
5. Fill out the PR template

### Code of Conduct

Please follow the [Openwater Code of Conduct](https://github.com/OpenwaterHealth/openwater-commons/blob/main/CODE_OF_CONDUCT.md).

## Questions?

- Open an issue for questions about what to add
- Tag maintainers for urgent matters
- Join discussions in the Openwater community

## Recognition

All contributors will be recognized in the Contributors section of the README.
